# AfriLION Grant Master Document Template

This document serves as the authoritative source for all grant applications (Lacuna Fund, Mozilla, Google.org, etc.).

## 1. Executive Summary
AfriLION is the first large-scale effort to build high-performance LLMs for African languages using a "Built IN Africa" decentralized data collection model.

## 2. Problem Statement
* **The Linguistic Gap:** 2,000+ languages, <0.1% of digital presence.
* **The Data Scarcity Wall:** Existing scraping methods fail for low-resource languages like Wolof, Shona, and Twi.
* **The Tokenization Tax:** Current models are 4-5x more expensive to run for African scripts.

## 3. Proposed Solution
* **The Wolof Pattern:** Shifting from "Reviewers" to "Paid Creators" in regional hubs (Dakar, Casablanca, Nairobi).
* **AfriLION Architecture:** 7B+ parameter models optimized for Ge'ez and Latin-based African scripts.
* **Open Source Commitment:** All datasets pushed to Hugging Face LocaleNLP org.

## 4. Implementation Plan (Phase 1)
* **Milestone 1:** Tokenizer optimization (95%+ parity with English compression).
* **Milestone 2:** Regional Coordinator recruitment.
* **Milestone 3:** 100M+ tokens of "Gold Standard" human-created data.

## 5. Budget & Stewardship
* **Human Capital:** 60% (Coordinators + Creators).
* **Compute:** 30% (TPU Research Cloud + H100 clusters).
* **Operations:** 10%.
